🚀 入境游管理系统 — 上线指南

从零开始，5步让你的4个同事都能在线使用。全程不需要写代码。

📑 目录（点击跳转）
  1. 注册 GitHub 账号
  2. 安装 GitHub Desktop 并上传代码
  3. 注册 Render 账号
  4. 部署系统到 Render
  5. 初始化系统 & 添加同事账号
1
注册 GitHub 账号
1
打开 GitHub 网站
在浏览器输入：https://github.com
💡 GitHub 就像是一个"代码网盘"，你把系统代码放上去，后面部署的时候 Render 会自动从那里拉取。
2
点击 Sign up 注册
右上角绿色按钮 Sign up
填写：
• Email：你的邮箱（常用邮箱）
• Password：设一个密码（至少15位，或8位含数字和字母）
• Username：起一个用户名（英文，如 lily-tour）
然后点 Create account
3
验证邮箱
GitHub 会给你的邮箱发一个验证码，打开邮箱，把那串数字填进去就行。
✅ 注册完成后，你会看到 GitHub 主页。不需要做任何其他操作，进入下一步。
2
安装 GitHub Desktop 并上传代码
💡 GitHub Desktop 是一个桌面软件，图形界面操作，比命令行简单得多。它会自动帮你安装 git。
1
下载 GitHub Desktop
在浏览器输入：https://desktop.github.com
点击页面中间的大按钮 Download for macOS（如果你是 Windows 就点 Windows 版）
2
安装并打开
下载完是一个 .dmg 文件（Mac）或 .exe 文件（Windows），双击安装。Mac 的话把 GitHub Desktop 图标拖到 Applications 文件夹里，然后打开它。
3
登录 GitHub 账号
打开 GitHub Desktop 后，会看到欢迎界面。点 "Sign in to GitHub.com"
• 会弹出浏览器页面
• 在浏览器里点 Authorize desktop（授权桌面应用）
• 回到 GitHub Desktop，确认登录成功
4
创建新仓库
在 GitHub Desktop 左上角，点 File → New repository...（或按快捷键 Cmd+N）
填写以下信息：
• Name：填 tour-system
• Local path：点右边的 Choose... 按钮，选择以下路径：
/Users/lily/Workbuddy/2026-07-02-11-15-10/tour-system-cloud
⚠️ 重要：不要勾选 "Initialize this repository with a README"，因为目录里已经有文件了！
• Git ignore：留空（目录里已有 .gitignore 文件）
• License：留空
• 点 Create repository
5
提交代码（Commit）
创建仓库后，GitHub Desktop 会显示所有文件。你需要在左下角做一次"提交"：
• 在左下角 Summary 框里填：初始提交
• 点蓝色的 Commit to main 按钮
• 等几秒钟，左边文件列表会变空（说明已提交成功）
6
发布到 GitHub（Push）
提交后，顶部会出现一个 "Publish repository" 按钮，点它！
• 弹出窗口，保持 "Keep this code private" 勾选（你的代码不需要公开）
• 点 Publish repository
• 等待上传完成（几秒钟）
✅ 完成后：打开浏览器访问 https://github.com/你的用户名/tour-system，能看到代码文件就说明上传成功了！
3
注册 Render 账号
💡 Render 是一个免费的服务器托管平台。你把 GitHub 上的代码告诉它，它会帮你运行，给你一个公网网址，别人就能访问了。
1
打开 Render 网站
在浏览器输入：https://render.com
2
用 GitHub 账号登录
点右上角 "Sign Up"，然后点 "GitHub" 图标登录（不用重新注册）
• 会跳转到 GitHub 授权页面
• 点 Authorize Render（授权 Render 访问你的 GitHub）
• 回到 Render，填写一些基本信息（名字、邮箱等）
• 不需要绑卡！免费版可以直接用
✅ 注册完成后，你会看到 Render 的 Dashboard（控制面板）。
4
部署系统到 Render
1
创建新的 Web Service
在 Render Dashboard 右上角，点 "New +" → 选 "Web Service"
2
连接 GitHub 仓库
页面会显示你的 GitHub 仓库列表，找到 tour-system，点右边的 "Connect"
⚠️ 如果看不到仓库，点 "Configure account" 链接，授权 Render 访问你的仓库。
3
填写部署配置
连接后进入配置页面，按以下内容填写：
• Name：tour-system（随意取，这会出现在网址里）
• Region：选 Singapore（离中国最近）或 Oregon
• Branch：main（默认就好）
• Root Directory：留空
• Runtime：选 Node
• Build Command：npm install
• Start Command：node server.js
• Instance Type：选 Free（免费版）
4
添加环境变量（重要！）
在配置页面往下滚动，找到 "Environment Variables" 区域，添加一项：
Key：JWT_SECRET
Value：填一串随机字符，比如 toursystem2026secretkey9988
💡 这个密钥用于加密登录信息。不设的话系统也能用，但服务器重启后所有人需要重新登录。
5
点击部署！
滑到页面最底部，点蓝色大按钮 Create Web Service
💡 部署需要 2-3 分钟。你会看到一个日志页面在滚动，等它出现 Server running on port 3000 就说明成功了！
6
获取公网网址
部署完成后，页面左上角会显示你的网址，类似：
https://tour-system-xxxx.onrender.com
✅ 就是这个网址！复制它，发给4个同事，他们打开就能用了！
5
初始化系统 & 添加同事账号
1
用管理员账号登录
打开你刚才拿到的公网网址，用默认管理员账号登录：
用户名：admin 密码：admin123
2
注入演示数据（可选）
登录后，点左侧菜单 「数据管理」，找到 「注入演示数据」 按钮，点击它。
💡 这会创建4个示例旅行团和一些报账数据，方便你先看看效果。正式使用时可以删除这些演示数据。
3
添加同事账号
在「数据管理」页面，找到 「用户管理」 区域，点 「添加用户」：
为每个同事创建账号：
• 用户名：英文，如 zhangsan
• 显示名：中文，如 张三
• 密码：至少6位
• 角色：选 普通成员（管理员才能管理用户）
点 创建
💡 创建4个同事的账号后，把各自的用户名和密码分别告诉他们。
4
修改管理员密码（重要！）
在「数据管理」→ 找到 「修改密码」，把默认的 admin123 改成一个你自己的密码。
⚠️ 一定要改！不然任何人都能用 admin123 登录管理员账号。
5
告诉同事们怎么用
把以下信息发给每个同事：
系统网址：https://tour-system-xxxx.onrender.com （替换成你实际的网址） 你的账号： 用户名：zhangsan 密码：xxxxxx 打开网址 → 输入用户名密码 → 登录就能用了！ 数据是自动同步的，你改了别人15秒内就能看到。

❓ 常见问题

Q: Render 免费版会丢数据吗？
A: 有可能。免费版如果长时间不访问，服务器会休眠，重启后文件数据可能丢失。建议每天在「数据管理」→「导出全部数据」备份一次。如果数据重要，可以考虑升级到 Render Starter 计划（$7/月），加上 Persistent Disk 就不会丢了。
Q: 同事打开网址后要等很久？
A: Render 免费版15分钟无访问会休眠，第一次打开需要等30秒左右唤醒。唤醒后就正常了。可以装一个定时访问工具（如 UptimeRobot）每5分钟ping一次来防止休眠。
Q: 以后代码更新了怎么办？
A: 在 GitHub Desktop 里修改文件 → 填写 Summary → Commit → 点 "Push origin"。Render 会自动检测到更新，自动重新部署。
Q: 4个人同时操作会冲突吗？
A: 系统有版本冲突检测。如果两人同时修改，后保存的人会看到提示"数据已被其他人更新"，点击刷新获取最新数据后再保存即可。正常使用不会丢数据。
Q: GitHub Desktop 里找不到目录怎么办？
A: 确保路径是对的。在 Mac 上，可以用 Finder → 前往 → 前往文件夹，粘贴路径确认目录存在。如果 Workbuddy 目录看不到，可能需要在 Finder 设置里显示隐藏文件（Cmd+Shift+.）。
